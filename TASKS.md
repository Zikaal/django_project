# План задач для Trello (Задачи 1 - 16)

## Выполненные задачи (Спринты 1-8)

### Задача 1. Базовая авторизация и управление профилями (Учет сотрудников)
**Описание:** 
Разработана система аутентификации (вход/выход) с использованием стандартных механизмов сессий Django. Созданы view и шаблоны (CRUD) для добавления, редактирования, просмотра и удаления пользователей (сотрудников) из системы. 
**Критерии приемки:**
- Настроен роутинг на `/accounts/login/`.
- Использованы встроенные или кастомные views для обработки логина/логаута.
- Созданы HTML-шаблоны панели управления списком сотрудников.

### Задача 2. Архитектура и структура дочерних компаний КМГ
**Описание:** 
Адаптирована структура базы данных под бизнес-реалии Группы Компаний. Разработаны модели для сущностей `OilCompany` (компании) и `Well` (буровые установки/скважины). Добавлен `oil_company` (foreign key) в модель `Profile`.
**Критерии приемки:**
- Создана модель `OilCompany` (name, region).
- Создана модель `Well` (name, type, max_drilling_depth, oil_company).
- Обновлен профиль пользователя.
- Описаны связи ORM: один-ко-многим (`ForeignKey`) между компанией и сотрудниками/скважинами.

### Задача 3. Панель Администратора (Django Admin)
**Описание:** 
Настроена мощная встроенная админка Django для управления справочниками и данными.
**Критерии приемки:**
- Зарегистрированы модели `OilCompany`, `Well`, `DailyProduction`, `Profile`.
- Настроены фильтры (list_filter), поиск (search_fields) и сортировка в админ-панели.

### Задача 4. Бизнес-логика: Журнал суточных рапортов добычи
**Описание:** 
Ключевой бизнес-функционал для хранения и расчета ежедневных производственных показателей работы скважин мастерами на местах.
**Критерии приемки:**
- Создана модель `DailyProduction` с уникальным ключом (`well` + `date`).
- Сделаны валидации: время работы 0-24ч, обводненность 0-100%, защита от двойного ввода рапорта по одной скважине в один день.
- Реализовано свойство/метод для расчета чистой нефти в тоннах по формуле: `Жидкость * (1 - Вода/100) * Плотность`.
- Разработана HTML-форма и сводная таблица рапортов с вычисляемыми колонками.

### Задача 5. Интеграция Tailwind CSS
**Описание:** 
Разработан и сверстан пользовательский интерфейс с использованием современного утилитарного фреймворка TailwindCSS.
**Критерии приемки:**
- Внедрен пакет `django-tailwind`.
- Созданы layout-файлы, боковая и верхняя навигация.
- Дизайн форм ввода и таблиц приведен к строгому корпоративному стилю.

### Задача 6. Контейнеризация и автоматизация окружения (Docker)
**Описание:** 
Внедрение оркестрации контейнеров для стандартизации разработки и деплоя. Новый сотрудник (или разработчик) должен иметь возможность поднять проект одной командой.
**Критерии приемки:**
- Написаны `Dockerfile` (с установкой Nodejs для Tailwind) и `docker-compose.yml`.
- Подняты сервисы Gunicorn (Django), Nginx, MySQL, Redis, Mailpit.
- Написан `Makefile` (команды `make up`, `make migrate`, `make shell`). Настройка локального окружения занимает не более 5 минут.
