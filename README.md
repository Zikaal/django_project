# Django Base Project

Базовый Django-проект с аутентификацией пользователей, расширенными профилями и модулем суточных рапортов по добыче скважин.

## Функционал

### Пользователи и профили
- регистрация пользователей
- вход и выход из системы
- автоматическое создание профиля для нового пользователя
- редактирование профиля
- просмотр списка пользователей
- доступ к защищенным страницам только для авторизованных пользователей
- управление профилями через Django Admin

### Суточные рапорты по добыче
- создание скважин
- создание суточных рапортов
- редактирование суточных рапортов
- удаление суточных рапортов
- просмотр списка всех рапортов
- автоматический расчет чистого дебита нефти
- валидация данных формы
- защита страниц рапортов через авторизацию
- управление скважинами и рапортами через Django Admin

## Установка и запуск проекта

### 1. Клонировать репозиторий
```bash
git clone https://github.com/USERNAME/REPOSITORY_NAME.git
cd REPOSITORY_NAME
```

### 2. Создать виртуальное окружение
```bash
python -m venv venv
```

### 3. Активировать виртуальное окружение

#### Windows
```bash
venv\Scripts\activate
```

#### macOS / Linux
```bash
source venv/bin/activate
```

### 4. Установить зависимости
```bash
pip install django
```

### 5. Создать и применить миграции
```bash
python manage.py makemigrations
python manage.py migrate
```

### 6. Создать суперпользователя
```bash
python manage.py createsuperuser
```

### 7. Запустить сервер
```bash
python manage.py runserver
```

После запуска проект будет доступен по адресу:

```bash
http://127.0.0.1:8000/
```

## Модели проекта

### Profile
Расширяет стандартную модель пользователя Django через связь `OneToOneField`.

Поля:
- `bio` — информация о пользователе
- `department` — отдел
- `phone_number` — номер телефона

Профиль создается автоматически при создании нового пользователя через Django Signals (`post_save`).

### Well
Модель скважины.

Поля:
- `name` — название скважины

### DailyProduction
Модель суточного рапорта по добыче.

Поля:
- `well` — связь со скважиной (`ForeignKey`)
- `date` — дата
- `work_time` — время работы, часов
- `liquid_debit` — дебит жидкости, м³/сут
- `water_cut` — обводненность, %
- `oil_density` — плотность нефти, т/м³

## Бизнес-логика

Для модели `DailyProduction` реализован расчет чистого дебита нефти по формуле:

```text
Дебит нефти = Дебит жидкости × (1 - Обводненность / 100) × Плотность нефти
```

Расчет выводится динамически через вычисляемое свойство модели.

## Валидация

Для суточных рапортов реализованы проверки:

- все поля обязательны
- `work_time` — от 0 до 24
- `water_cut` — от 0 до 100
- нельзя создать две записи для одной и той же скважины на одну и ту же дату

Уникальность обеспечивается:
- на уровне формы
- на уровне базы данных через `UniqueConstraint`

## CRUD-функционал

Для модели `DailyProduction` реализованы:

- `CreateView` — создание записи
- `ListView` — список записей
- `UpdateView` — редактирование записи
- `DeleteView` — удаление записи

## Доступ

Следующие страницы доступны только авторизованным пользователям:

- список пользователей
- страница профиля
- список рапортов
- создание рапорта
- редактирование рапорта
- удаление рапорта

Защита реализована через `LoginRequiredMixin`.

## Админ-панель

В Django Admin зарегистрированы следующие модели:

- `Profile`
- `Well`
- `DailyProduction`

Через админ-панель можно просматривать, добавлять, редактировать и удалять записи.

## Проверка проекта

Перед сдачей проекта стоит проверить:

- миграции применяются без ошибок
- регистрация и вход работают
- профиль пользователя создается автоматически
- редактирование профиля работает
- список пользователей отображается
- рапорт создается через форму
- валидация не позволяет ввести неверные данные
- расчет нефти отображается корректно
- редактирование и удаление рапортов работают
- неавторизованный пользователь не может открыть защищенные страницы
- модели доступны в `/admin`
